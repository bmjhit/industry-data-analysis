# A 股行业与基金投资看板

这是一个本地静态看板原型，用于按日、周、月、季度、年度观察 A 股行业表现，并把行业强弱、估值、资金、风险与基金投资赛道建议联系起来。

## 当前版本

- 页面由本地轻量服务托管，用于保存私有买入记录和触发数据更新；无后端数据库。
- 优先使用 `data/industry-live.json` 真实缓存数据，失败时回落到 `data/industry-sample.json` 示例数据。
- 支持行业总览、周期切换、行业详情、基金赛道建议、组合配置建议。
- 所有输出均为研究辅助，不构成保证收益或确定性投资建议。

## 使用方式

安装依赖并生成真实数据：

```bash
python3 -m pip install -r requirements.txt
python3 scripts/update_data.py --limit 12 --funds-per-industry 30 --fund-scan-limit 120 --exposure-threshold 2.5
python3 scripts/backtest_short_term.py
```

启动本地看板服务：

```bash
python3 scripts/serve_dashboard.py 4174
```

然后打开 `http://localhost:4174/`。页面会优先读取 `data/industry-live.json`，如果该文件不存在或读取失败，会回落到 `data/industry-sample.json` 示例数据。

页面默认缓存容量为每行业最多 30 只基金。“每行业候选”只控制当前展示数量，修改后会通过本地 `/api/candidate-view` 自动切换已缓存候选，也可点“应用缓存”再次载入；“更新缓存”始终按 30 只/行业重新执行真实数据更新和短线回测，用于补齐缓存或更新行情。若仅用 `python3 -m http.server` 启动，页面仍可浏览，但无法调用这两个接口。

## 后续数据接入方向

- A 股股票与行业行情：当前通过 AkShare 获取东方财富公开行业板块、历史行情和成分股数据。
- 基金数据：当前通过 AkShare 获取东方财富开放式基金排行数据，并按细分行业到基金赛道关键词、持仓穿透两条路径生成候选基金。
- 候选池扩容：`--funds-per-industry` 控制每个行业保留候选数，`--fund-scan-limit` 控制持仓穿透扫描的基金数，`--exposure-threshold` 控制行业持仓暴露阈值。
- 个基量化评分：基于历史净值计算 20/60/120 日动量、年化波动、最大回撤、近 60 日胜率和类 Sharpe 指标，生成上涨概率评分、风险分和综合分。
- 基金质量过滤：加入基金规模、成立时间、基金经理任期、持仓集中度、跟踪误差等过滤与扣分项。
- 筛选摘要：展示候选基金总数、通过数量、淘汰数量、数据不足数量和主要过滤原因。
- 风险偏好联动：稳健型、均衡型、进取型会改变候选基金过滤阈值、今日推荐和建议持仓周期。
- 持仓穿透匹配：将基金重仓股与行业成分股交叉，补充基金对行业的真实暴露比例；行情接口失败时会用缓存中的重点企业做轻量穿透。
- 短线观察池：基于 20 日动量、60 日胜率、风险分、最大回撤、行业强度和质量过滤，筛选 1-6 周短期观察基金，并给出复核纪律。
- 短线模型回测：基于历史净值逐点生成信号，统计未来 5/10/20 个净值点收益、胜率和回撤。
- 买入记录与策略验证：本地保存基金代码、买入日期和金额；每日访问页面或更新缓存时按最新净值评估盈亏，并统计买入当日实际推荐信号在后续 10 个净值点的成功率。
- 策略优化观察：成熟推荐样本达到最低数量后，比较低风险、高上涨评分样本的命中率，只提供阈值观察建议，不基于少量交易自动修改策略。
- 后续可补充：基金规模、跟踪指数、基金经理、最大回撤、同类排名、申万行业分类。

## 投资原则

- 先做资产配置，再选具体基金。
- 核心仓优先宽基和低费率指数基金。
- 行业主题基金作为卫星仓，控制比例和回撤。
- 主动基金重点观察基金经理任期、风格稳定性、回撤控制和长期超额收益。
